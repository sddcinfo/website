End-to-End Execution Guide

From bare metal to a running cluster in four steps.

Step 1: Configure Your Environment

Before running any playbooks, you must configure the variables to match your specific hardware and environment in the ansible-provisioning-server and ansible-kubernetes-nodes projects.

Step 2: Run the Provisioning Server Playbook

This command configures your central server. It only needs to be run once, or whenever you update its configuration.

cd /path/to/ansible-provisioning-server
ansible-playbook -i inventory site.yml

Step 3: Boot the Bare-Metal Nodes

Power on your bare-metal servers and ensure they are set to PXE boot. The automated process will now take over. You can monitor this entire process by opening a web browser and navigating to the IP of your provisioning server (e.g., http://10.10.1.1).

Step 4: Run the Kubernetes Cluster Playbook

Once the status page shows your nodes as "DONE", you can proceed with configuring them as a Kubernetes cluster.

cd /path/to/ansible-kubernetes-nodes
ansible-playbook -i inventory.ini site.yml

After this playbook completes, your Kubernetes cluster will be fully operational.
