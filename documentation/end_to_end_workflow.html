End-to-End Execution Guide

From bare metal to a running cluster.

Step 1: Configure Your Environment

Before running any playbooks, you must configure the variables to match your specific hardware and environment.

  1. Configure Node Details: Edit ansible-provisioning-server/roles/netboot/vars/main.yml to define the MAC addresses, IPs, and hostnames for your machines.
  2. Configure Cluster Details: Edit ansible-kubernetes-nodes/group_vars/all.yml to set the Kubernetes version and Ceph devices.
  3. Define Inventory: Edit ansible-kubernetes-nodes/inventory.ini to assign nodes to controller and worker groups.

Step 2: Run the Provisioning Server Playbook

This command configures your central server. The first time it runs, it will generate new SSH keys for automation.

cd /path/to/ansible-provisioning-server
ansible-playbook -i inventory site.yml

Step 3: Distribute SSH Keys

This is a critical one-time step. After the provisioning server is set up, copy the public SSH keys it generated to the Kubernetes cluster project.

  1. Copy the content of /home/sysadmin/.ssh/sysadmin_automation_key.pub from the provisioning server.
  2. Paste it into the sysadmin_ssh_pub_key variable in ansible-kubernetes-nodes/group_vars/all.yml.
  3. Copy the content of /root/.ssh/root_automation_key.pub from the provisioning server.
  4. Paste it into the root_ssh_pub_key variable in ansible-kubernetes-nodes/group_vars/all.yml.

Step 4: Boot the Bare-Metal Nodes

Power on your servers. They will automatically PXE boot and install Ubuntu. Monitor the progress on the live status page at http://10.10.1.1.

Step 5: Run the Kubernetes Cluster Playbook

Once the status page shows your nodes as "DONE", run the final playbook to configure the cluster.

cd /path/to/ansible-kubernetes-nodes
ansible-playbook -i inventory.ini site.yml

After this playbook completes, your Kubernetes cluster will be fully operational.
